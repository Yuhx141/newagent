# AI-Driven Algorithm Solver - 项目文档

## 一、项目概述

### 1.1 项目名称
AI-Driven Algorithm Solver（AI 驱动的算法解题助手）

### 1.2 项目定位
这是一个基于大语言模型的智能算法解题系统，能够自动理解题目、设计算法、生成代码并提供详细注释。

### 1.3 核心痛点解决

#### 传统算法学习的痛点：
1. **题目理解困难**：复杂题意难以快速抓住关键信息
2. **思路形成缓慢**：缺乏系统性的算法选择方法
3. **代码调试低效**：错误定位困难，反复试错
4. **知识迁移困难**：做过题目不会举一反三

#### 本项目的解决方案：
1. **智能题目解析**：自动提取数据规模、时间复杂度要求、输入输出格式
2. **算法策略推荐**：基于题目特征匹配最优算法（DP、贪心、图论等）
3. **代码自动生成**：生成规范、高效、带详细注释的代码
4. **思路可视化**：展示完整的思考过程和算法选择理由

## 二、核心逻辑流

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│              AI-Driven Algorithm Solver              │
└─────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│  题目解析模块  │  │  算法设计模块  │  │  代码生成模块  │
│  (Parser)     │  │  (Designer)   │  │  (Generator)  │

        │                 │                 │
        └─────────────────┼─────────────────┘
                          │
                          ▼
                ┌─────────────────┐
                │  验证优化模块    │
                │  (Validator)    │
                └─────────────────┘
```

### 2.2 模块详解

#### 模块 1：题目解析模块 (Parser)
**功能**：
- 提取题目中的关键信息
- 识别数据规模（n 的范围、数值范围）
- 分析时间复杂度要求
- 理解输入输出格式

**长链推理示例**：
```
输入：给定一个长度为 n 的数组，找出其中第 k 大的元素 (1 ≤ n ≤ 10^5)

推理链：
1. 识别数据规模：n 最大 10^5
2. 推断时间复杂度：O(n²) 会超时，需要 O(n) 或 O(n log n)
3. 识别问题类型：Top-K 问题
4. 候选算法：快速选择、堆排序、归并排序
```

#### 模块 2：算法设计模块 (Designer)
**功能**：
- 基于题目特征匹配算法模板
- 进行多轮推理选择最优解
- 生成详细的解题思路

**决策树示例**：
```
问题类型判断：
├─ 最值问题？
│  ├─ 连续子数组 → 动态规划 (Kadane 算法)
│  └─ 非连续 → 贪心/DP
├─ 查找问题？
│  ├─ 有序数组 → 二分查找
│  └─ 无序数组 → 哈希表/排序
├─ 图论问题？
│  ├─ 最短路径 → BFS/Dijkstra
│  └─ 连通性 → DFS/并查集
└─ 字符串问题？
   ├─ 模式匹配 → KMP
   └─ 子序列 → DP/双指针
```

#### 模块 3：代码生成模块 (Generator)
**功能**：
- 生成符合规范的 C++ 代码
- 添加详细注释（函数功能、参数说明、返回值）
- 遵循最佳实践（ios 加速、vector 替代数组）

**代码规范**：
```cpp
/*
 * 函数功能：清晰描述函数作用
 * 参数：
 *   - param1: 参数 1 说明
 *   - param2: 参数 2 说明
 * 返回值：返回值说明
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */
ReturnType functionName(ParamType1 param1, ParamType2 param2) {
    // 关键步骤注释
    ...
}
```

#### 模块 4：验证优化模块 (Validator)
**功能**：
- 边界条件检查
- 时间复杂度验证
- 空间复杂度优化
- 测试用例生成

## 三、技术特色

### 3.1 长链推理能力
- 能够从题目描述推理到算法选择
- 多层决策：问题类型 → 算法类别 → 具体实现
- 复杂度分析与优化建议

### 3.2 多 Agent 协作
虽然本项目是单 Agent 实现，但架构设计支持扩展为多 Agent：
- **Parser Agent**：专职题目解析
- **Designer Agent**：专职算法设计
- **Coder Agent**：专职代码生成
- **Reviewer Agent**：专职代码审查

### 3.3 知识图谱
内置算法知识图谱：
- 100+ 经典算法模板
- 算法适用场景索引
- 复杂度对比矩阵

## 六、使用指南

### 6.1 编译运行

```bash
# 编译单个题解
g++ -std=c++17 -O2 solutions/001_two_sum.cpp -o two_sum

# 运行
./two_sum
```

### 6.2 输入输出示例

以 `001_two_sum.cpp` 为例：
```bash
# 输入
4 9
2 7 11 15

# 输出
0 1
```

### 6.3 添加新题解

1. 在 `solutions/` 目录创建新文件
2. 按照统一格式编写代码
3. 包含详细注释和测试样例
